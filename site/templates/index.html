01

The breach

Your organisation opened Vertex AI to the team. Nobody is watching what goes in or comes out.

Sensitive data leaves your network in plain text. An engineer pastes a stack trace containing your entire customer table. A support agent includes account numbers in a prompt. A developer copies a production private key into a code review request. All of it travels to Google's API unscreened.

Responses come back unchecked. The model can generate social security numbers, credit card data, credentials, or harmful content. Without a proxy, that output goes straight to the user's screen with no filter, no warning, no record.

Model Armor has documented gaps. It does not enforce on streaming endpoints. It only works with Gemini models, not third-party ones. It fails open when unavailable, passing unscreened traffic by design. Cloud DLP exists but nothing wires it into the request path.

Library-based screening is a policy, not a control. If screening lives in application code, a misconfigured client bypasses it. If it is a shared library, a developer can skip the import. A network proxy enforces at the infrastructure layer. Clients cannot opt out.

What happens without a proxy

  • ● Customer PII in Vertex AI audit logs, sent by an engineer who pasted a support ticket
  • ● Production private keys travelling to Google in plain text inside a prompt
  • ● AWS root credentials logged in Vertex AI request history
  • ● Streaming endpoints bypassing Model Armor entirely
  • ● Zero screening on third-party models (Anthropic, Llama) running in Vertex AI
  • ● Model Armor skipping sanitization when overloaded, passing traffic through

The garrison

Bulwarkai is infrastructure, not a library. It runs between your users and Vertex AI. Every request passes through the inspector chain before reaching the model. Every response passes through before reaching the user. No client can skip it.

Drop-in compatibility

Point your existing tools at Bulwarkai. Anthropic, OpenAI, and Vertex AI formats all work. No client changes needed.

POST /v1/messages

Anthropic Messages

Works with the Anthropic SDK and Claude Code. Set base_url to your proxy endpoint.

POST /v1/chat/completions

OpenAI Chat Completions

Works with the OpenAI SDK and any tool that speaks the OpenAI format.

/models/{model}:generateContent

Vertex AI native

Pass-through for Gemini native format. Streaming and non-streaming.

GET /v1/models

Model discovery

Returns available models in OpenAI format so SDKs can enumerate them.

04

Deploy in 30 seconds

Demo mode runs the full screening pipeline with canned responses. No GCP account, no Vertex AI, no cost.

# Clone and run
git clone https://github.com/Daviey/bulwarkai.git
cd bulwarkai
DEMO_MODE=true LOCAL_MODE=true go run .

# In another terminal, send a test prompt with a fake SSN
curl -s http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"gemini-2.5-flash","messages":[
    {"role":"user","content":"BULWARKAI-TEST-SSN-000-00-0000"}
  ]}'

# {"error":{"message":"Blocked by bulwarkai: SSN detected",...}}
